Security

Last updated: March 12, 2026

Model Pipeline AI Inc. is committed to protecting the security of the ImagePipeline platform and the data of our users. We implement industry-standard security practices across our infrastructure, application layer, and data handling processes.

1. Infrastructure Security

Our platform is built on secure, enterprise-grade cloud infrastructure. Key controls include:

  • Encrypted communication over HTTPS/TLS for all data in transit
  • Secure cloud infrastructure with strict network access controls
  • API authentication and key management systems
  • Rate limiting and automated abuse detection
  • Continuous monitoring and alerting for suspicious activity

2. Data Protection

User data is protected through encryption at rest, secure storage practices, and role-based access controls that limit data access to authorised personnel only.

Generated images and prompts are stored temporarily and automatically deleted within 30 days of creation, as described in our Privacy Policy. We do not use user-submitted content to train AI models.

3. Responsible Disclosure

If you discover a security vulnerability in ImagePipeline, please report it responsibly. Do not exploit the vulnerability or share details publicly before giving us the opportunity to address it.

Report security issues to: security@imagepipeline.io

We will acknowledge receipt of your report promptly and work with you to understand and resolve the issue. We appreciate the efforts of security researchers who help keep our platform safe.
